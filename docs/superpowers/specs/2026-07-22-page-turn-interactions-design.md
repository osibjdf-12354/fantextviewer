# Page Turn Interactions Design

## Goal

페이지 넘김 모드에서 좌우·상하·양방향 스와이프와 상하단 탭 넘김을 제공하고, 120Hz 기기의 전환 끊김과 페이지 상·하단 레이아웃 문제를 고친다. 기존 텍스트 선택, 검색, 북마크, 위치 복원은 유지한다.

## Confirmed Behavior

- 표시 설정에 `좌우 넘김`, `상하 넘김`, `둘 다`를 둔다.
- 기존 저장 데이터와 새 설치의 기본값은 `좌우 넘김`이다.
- 왼쪽 또는 위쪽 스와이프는 다음 페이지로 이동한다.
- 오른쪽 또는 아래쪽 스와이프는 이전 페이지로 이동한다.
- `둘 다`에서는 이동 임계점을 넘은 뒤 우세한 축을 선택하고, 해당 축으로 페이지가 손가락을 따라간다.
- 본문 위쪽 절반을 탭하면 이전 페이지, 아래쪽 절반을 탭하면 다음 페이지로 이동한다.
- 길게 누르기는 페이지 넘김을 취소하고 기존 `SelectableText` 선택 동작에 맡긴다.
- 첫 페이지의 이전 동작과 마지막 페이지의 다음 동작은 아무 변화 없이 끝난다.

## Root Causes in the Current Code

- 페이지 화면은 `PageView`의 기본 가로축만 사용하므로 세로 전환 경로가 없다.
- 페이지 본문에 탭 처리기가 없어 탭으로 페이지를 바꿀 수 없다.
- Android 창이 높은 주사율을 명시적으로 요청하지 않는다.
- 페이지 계산은 본문 전체 높이를 사용하지만 페이지 번호 표시는 그 위에 겹쳐 그 영역의 글자가 가려진다.
- 페이지 경계가 줄바꿈 바로 앞에서 끝날 수 있어 다음 페이지가 줄바꿈으로 시작한다.

## Settings and Persistence

`PageTurnDirection` 열거형을 `horizontal`, `vertical`, `both` 값으로 추가하고 `ReaderSettings`에 저장한다. JSON 키가 없는 기존 설정은 `horizontal`로 복원한다. 기존 표시 설정의 초안·적용·저장 흐름을 그대로 사용하며 새 의존성은 추가하지 않는다.

## Page Turn Component

가로 `PageView`를 현재 인덱스를 외부에서 받는 작은 2축 페이지 전환 컴포넌트로 교체한다. 리더는 페이지 목록, 현재 인덱스, 문자 오프셋을 계속 소유한다. 전환 컴포넌트는 현재 페이지와 이동 방향의 인접 페이지만 그려 제스처와 전환 애니메이션을 담당한다.

검색, 북마크, 페이지 번호 이동, 저장 위치 복원은 리더의 현재 인덱스를 즉시 바꾸는 기존 의미를 유지한다. 사용자가 전환을 끝냈을 때만 리더에 새 인덱스를 알리고, 리더는 해당 페이지 시작 오프셋을 저장한다. 페이지 계산 배치가 추가되거나 정확한 전체 페이지 목록으로 교체되면 현재 문자 오프셋으로 인덱스를 다시 결정한다.

## Pointer and Selection Handling

전환 컴포넌트는 원시 포인터 이동을 관찰하되 자식의 포인터 이벤트를 막지 않는다. 짧고 이동량이 작은 입력만 탭으로 판정한다. 길게 누른 뒤의 이동은 페이지 제스처로 시작하지 않아 `SelectableText`가 선택을 처리할 수 있다. 여러 손가락이 닿거나 축이 확정되기 전에 취소된 입력은 페이지를 바꾸지 않는다.

`horizontal`과 `vertical`은 선택한 축만 받는다. `both`는 임계점을 넘은 최초 이동의 절댓값이 더 큰 축을 해당 포인터의 축으로 고정한다. 이동 거리가 충분하거나 해당 방향 속도가 임계값을 넘으면 페이지를 완료하고, 그렇지 않으면 원래 페이지로 돌아간다.

## Animation and Refresh Rate

드래그 위치와 완료·복귀 애니메이션은 Flutter `AnimationController`의 vsync로 구동한다. 고정 간격 타이머나 프레임 수 기반 애니메이션은 사용하지 않는다.

Android `MainActivity`는 창이 표시될 때 현재 디스플레이가 지원하는 최대 주사율을 `preferredRefreshRate`로 요청한다. 이는 선호도이며 배터리 절약, 발열, 제조사 정책에 따라 시스템이 더 낮은 주사율을 선택할 수 있다. 별도 Flutter 플러그인은 추가하지 않는다.

## Pagination and Layout

페이지 모드에서만 본문 계산 높이와 렌더링 높이 양쪽에서 하단 40 논리 픽셀을 페이지 번호 영역으로 예약한다. 페이지 번호는 지금처럼 오른쪽 아래에 표시하지만 본문은 예약 영역 위에서 끝난다. 같은 크기를 계산과 렌더링에 사용해 페이지 범위와 실제 화면을 일치시킨다.

페이지 끝 위치가 줄바꿈 문자 바로 앞이면 그 줄바꿈 하나를 이전 페이지 범위에 포함한다. 따라서 다음 페이지는 의도치 않은 경계 줄바꿈으로 시작하지 않는다. 연속된 줄바꿈 중 나머지는 실제 빈 줄로 보존한다. 페이지 범위는 여전히 원문을 중복이나 누락 없이 모두 덮고 서로 맞닿아야 한다.

페이지 계산 캐시의 알고리즘 버전을 올려 이전 화면 높이와 경계 규칙으로 만든 캐시를 재사용하지 않는다.

## Error and Boundary Behavior

- 전환 중 페이지 목록이나 설정이 바뀌면 진행 중 애니메이션을 취소하고 현재 문자 오프셋에 맞는 페이지를 표시한다.
- 첫·마지막 페이지 바깥 동작은 오프셋과 저장 상태를 바꾸지 않는다.
- 페이지 계산 중 아직 존재하지 않는 다음 페이지로 넘기려는 입력은 무시하고 기존 계산을 계속한다.
- Android의 주사율 요청 실패나 시스템 제한은 앱 실행을 막지 않는다.

## Verification

- 설정의 세 방향 JSON 왕복과 누락 값의 가로 기본값을 검사한다.
- 좌우, 상하, 양방향 스와이프가 올바른 인접 페이지로 이동하는지 검사한다.
- 위쪽·아래쪽 탭, 첫·마지막 페이지 경계, 긴 누르기 시 페이지 유지 동작을 검사한다.
- 줄바꿈 경계 뒤 페이지가 불필요한 빈 줄로 시작하지 않고 모든 페이지가 원문을 정확히 덮는지 검사한다.
- 페이지 계산 크기와 렌더링 하단 여백이 같은 40픽셀을 사용하는지 검사한다.
- 전체 `flutter analyze`, `flutter test`, 릴리스 APK 빌드를 실행한다.
- 실제 120Hz 선택 여부와 체감 전환은 지원 Android 기기의 릴리스 빌드에서 최종 확인한다.

## Out of Scope

- 페이지 넘김 효과 종류나 속도 사용자 설정
- 3D 책장 넘김, 그림자, 탄성 효과
- 스크롤 모드의 입력·레이아웃 변경
- Android 시스템이 강제한 주사율 제한 우회
