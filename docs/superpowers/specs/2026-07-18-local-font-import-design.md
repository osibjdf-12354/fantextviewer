# 로컬 글꼴 가져오기 설계

## 목표

햄버거 메뉴의 `표시 설정`에서 Android 파일 선택기로 TTF/OTF 글꼴을 가져오고, 여러 글꼴을 선택하거나 삭제할 수 있게 한다. 가져온 파일은 앱 내부 저장소에 복사하여 원본이 이동되거나 삭제되어도 계속 사용할 수 있게 한다.

## 범위

- 시스템 기본 글꼴과 가져온 글꼴 중 하나를 본문 글꼴로 선택한다.
- 가져온 글꼴은 앱을 다시 실행해도 목록과 선택 상태가 유지된다.
- 글꼴 변경 시 본문, 설정 미리보기, 페이지 계산에 같은 글꼴을 사용한다.
- 인터넷 글꼴 다운로드, 글꼴 내부 메타데이터 파싱, 굵기·기울임꼴 조합은 추가하지 않는다.

## 저장 구조

애플리케이션 지원 디렉터리 아래 `fonts` 폴더를 가져온 글꼴 목록의 단일 원본으로 사용한다. 별도 글꼴 목록이나 데이터베이스는 저장하지 않는다. `ReaderSettings`에는 선택된 글꼴의 파일명만 nullable 값으로 저장하며, null은 시스템 기본 글꼴을 뜻한다.

가져올 때 원본 파일명을 유지한다. 같은 이름이 이미 있으면 확장자 앞에 순번을 붙여 기존 파일을 덮어쓰지 않는다. 화면에는 확장자를 제외한 파일명을 표시한다. Flutter에 등록할 글꼴 family 값은 저장된 파일명에서 안정적으로 파생한다.

## 구성요소와 데이터 흐름

작은 글꼴 저장소 구성요소가 다음 작업만 담당한다.

- `fonts` 폴더의 TTF/OTF 파일 조회
- 선택한 파일의 확장자 확인, 고유 이름 결정, 앱 저장소 복사
- Flutter `FontLoader` 등록
- 앱 내부 복사본 삭제

앱은 저장된 글꼴이 선택된 상태로 리더를 열 때 해당 파일을 먼저 등록한다. 표시 설정에서 기존 글꼴을 선택하거나 새 글꼴을 가져올 때도 적용 전에 등록한다. 등록에 성공한 글꼴의 family를 본문 `TextStyle`과 미리보기 `TextStyle`에 전달한다.

페이지 캐시 서명에는 선택된 글꼴 파일명을 포함한다. 글꼴이 바뀌면 기존 표시 설정 변경과 같은 경로로 페이지 상태를 무효화하고 현재 문자 오프셋을 기준으로 다시 계산한다.

## 표시 설정

기존 `표시 설정` 바텀시트에 `글꼴` 영역을 추가한다.

- 첫 항목은 `시스템 기본 글꼴`이다.
- 그 아래에 가져온 글꼴을 파일명 기준으로 정렬해 표시한다.
- `로컬 글꼴 가져오기` 버튼은 기존 `file_selector`로 `.ttf`, `.otf` 파일 하나를 선택한다.
- 가져오기에 성공한 글꼴은 즉시 초안의 선택 글꼴이 된다.
- 각 가져온 글꼴에는 접근 가능한 삭제 버튼과 확인 대화상자를 둔다.
- 삭제 대상이 현재 저장 설정 또는 초안에서 선택된 글꼴이면 둘 모두 시스템 기본 글꼴로 되돌리고 저장 설정은 즉시 저장한다.
- `적용` 전에는 선택 변경만 초안에 머물지만, 가져오기와 삭제는 파일 작업이므로 즉시 반영한다.

## 오류 처리

- TTF/OTF가 아닌 파일은 복사하지 않고 지원 형식을 안내한다.
- 복사 또는 `FontLoader` 등록에 실패하면 새 복사본을 제거하고 한국어 오류를 표시한다.
- 저장 설정이 가리키는 글꼴 파일이 없거나 시작 시 등록에 실패하면 시스템 기본 글꼴로 복구한다.
- 삭제는 앱 내부 복사본만 대상으로 하며 원본 파일은 변경하지 않는다.
- 파일 선택 취소는 오류로 표시하지 않는다.

## 테스트

TDD로 다음 동작을 먼저 실패하는 테스트로 고정한다.

- 선택 글꼴 파일명의 JSON 저장과 복원
- 임시 디렉터리에서 TTF/OTF 목록, 중복 이름 복사, 삭제
- 잘못된 확장자와 등록 실패 시 복사본을 남기지 않는 동작
- 표시 설정에서 시스템 기본 글꼴과 가져온 글꼴 선택
- 가져온 글꼴 삭제 시 시스템 기본 글꼴 복구
- 본문과 미리보기의 `fontFamily` 적용
- 페이지 캐시 서명에 선택 글꼴이 포함되어 글꼴 변경 시 재계산되는 동작

## 완료 기준

사용자는 햄버거 메뉴의 `표시 설정`에서 로컬 TTF/OTF를 가져와 바로 적용하고, 앱 재실행 뒤에도 사용하며, 목록에서 안전하게 삭제할 수 있다. 원본 파일 이동·삭제는 가져온 글꼴에 영향을 주지 않고, 실패한 가져오기는 설정이나 앱 저장소에 찌꺼기를 남기지 않는다.
