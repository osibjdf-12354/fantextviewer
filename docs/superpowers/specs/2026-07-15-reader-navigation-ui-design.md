# 글봄 페이지 이동과 읽기 UI 개선 설계

## 목표

스크롤 읽기에서도 정확한 페이지 이동과 페이지 기반 북마크를 제공하고, 읽기 화면과 표시 설정의 조작을 단순하게 만든다.

## 페이지 계산과 이동

- 현재 `TextPainter` 기반 페이지 계산을 두 읽기 모드에서 공용으로 실행한다.
- 페이지 크기는 현재 화면, 글자 크기, 줄 간격, 좌우 여백으로 계산한다.
- 위치 이동 입력은 정수 페이지 번호만 허용한다. 퍼센트 입력은 제거한다.
- 계산 중에는 `페이지를 계산하고 있습니다.`를 표시한다.
- 범위를 벗어난 입력에는 `1~N 사이 페이지를 입력해 주세요.`를 표시한다.
- 글자 크기, 줄 간격, 여백 또는 화면 크기가 바뀌면 기존 계산을 취소하고 다시 계산한다.

## 읽기 화면과 종료

- 읽기 화면 상단바 높이를 기본 56px에서 48px로 줄인다. 햄버거와 북마크 버튼의 48px 터치 영역은 유지한다.
- 페이지 모드의 하단 페이지 번호와 슬라이더 영역을 제거한다.
- 페이지 본문은 제거된 하단 영역까지 사용한다.
- 햄버거 메뉴에 `앱 종료` 항목을 추가하고 Android 앱을 종료한다.

## 북마크

- 북마크 저장 형식은 기존 원문 오프셋을 유지한다. 이 방식이 글꼴이나 화면 크기가 바뀌어도 원문의 같은 위치를 보존한다.
- 북마크 목록과 저장 안내는 퍼센트 대신 현재 페이지 번호를 표시한다.
- 북마크 선택은 저장된 오프셋에 대응하는 페이지 또는 스크롤 위치로 이동한다.

## 색상 템플릿

- 텍스트 이름 버튼을 제거하고 배경색과 글자색을 함께 보여주는 색상 견본 버튼으로 바꾼다.
- 각 견본은 `가` 한 글자로 전경색을 보여주며, 접근성 툴팁에는 기존 템플릿 이름을 유지한다.
- RGB 직접 입력과 기본 배경색 `R196 G236 B187`은 유지한다.

## 표시 설정 단계 버튼

- 기존 세 슬라이더를 왼쪽 감소 버튼, 현재 값, 오른쪽 증가 버튼으로 교체한다.
- 글자 크기는 `14~36`, `1` 단위로 조절한다.
- 좌우 여백은 `8~40`, `1` 단위로 조절한다.
- 줄 간격은 `1.2~2.2`, `0.1` 단위로 조절한다.
- 최솟값과 최댓값에서는 해당 방향 버튼을 비활성화한다.
- 기존에 저장된 글자 크기와 여백의 소수값은 가장 가까운 정수로, 줄 간격은 소수 첫째 자리로 정규화한다.

## 구현 범위

- 기존 `ReaderView`, 페이지 계산기, 저장 모델을 재사용한다.
- 새 패키지나 별도 상태 관리 계층은 추가하지 않는다.
- 저장 데이터 형식은 변경하지 않아 기존 읽기 위치와 북마크를 그대로 사용한다.

## 검증

- 스크롤 모드에서 페이지 계산 후 정수 페이지 이동이 성공하는 위젯 테스트
- 퍼센트 입력 제거와 페이지 범위 오류 테스트
- 하단바 제거, 종료 메뉴, 페이지 기반 북마크 표시 테스트
- 색상 견본의 색과 접근성 툴팁 테스트
- 단계 버튼의 단위, 범위 제한, 과거 소수값 정규화 테스트
- 기존 대용량 파일, 페이지 계산 취소, 한글 인코딩, 저장 복구 테스트 전체 회귀 실행
