# 판갤텍뷰 잔여 문제 해결 설계

## 목표

현재 확인된 파일 가져오기, 상태 무결성, 복구, 페이지 오류 처리 문제를 해결하고 사용자에게 보이는 `geulbom`/`글봄` 명칭을 `fantextviewer`/`판갤텍뷰`로 바꾼다.

기존 설치 위 업데이트와 앱 데이터 보존을 위해 Android `applicationId=com.songs.geulbom`과 기존 사용자의 서명 키는 호환성 식별자로 유지한다.

## 파일 접근

내부 공유 저장소의 폴더 탐색은 현재 직접 경로 방식을 유지한다. 이 경로는 개인 사이드로드 앱에서 사용자가 허용한 `MANAGE_EXTERNAL_STORAGE`에 의존하며, SD 카드·USB·클라우드의 폴더 탐색은 지원 범위에 포함하지 않는다.

SD 카드·USB·클라우드를 포함한 시스템 단일 파일 선택은 Android 네이티브 `ACTION_OPEN_DOCUMENT`로 구현한다. 선택한 파일은 네이티브 스트림으로 앱의 `filesDir/imported_texts`에 원자적으로 복사하고, Dart에는 전체 바이트나 임시 캐시 경로가 아니라 영구 복사본 경로만 반환한다. 같은 URI를 다시 선택하면 같은 앱 내부 경로를 갱신한다. 복사 중 64MiB를 넘으면 즉시 중단하고 임시 파일을 삭제한다.

기존 `file_selector` 캐시 경로가 최근 문서에 남아 있으면 파일이 존재하는 동안 앱 내부 가져오기 디렉터리로 승격하고 해당 문서 상태의 키를 새 경로로 이동한다. 이미 Android가 캐시를 삭제한 파일은 복구할 수 없으므로 기존의 파일 재선택 안내를 사용한다.

## 문서 로드와 상태

문서 상태 변경은 파일 디코딩과 읽기 전후 검증이 성공한 뒤에만 적용한다. 로드 순서는 다음과 같다.

1. 기존 문서 상태와 파일 메타데이터를 읽는다.
2. 저장된 인코딩 또는 사용자가 선택한 임시 인코딩으로 디코딩한다.
3. 내용 SHA-256과 읽기 전후 메타데이터를 검증한다.
4. 내용이 교체된 경우 자동 인코딩으로 한 번 다시 디코딩한다.
5. 파일 지문, 인코딩, 최근 문서, 진행 위치를 한 번에 갱신한다.
6. 본문을 표시한다.
7. 상태 저장을 시도하되 실패해도 표시한 본문은 유지하고 오류를 알린다.

`DocumentState`에 내용 SHA-256을 저장하고 스키마 버전을 올린다. 기존 상태에는 내용 지문이 없으므로 첫 성공 로드에서는 크기·수정일 기준 동작을 유지하고 지문만 채운다. 이후에는 내용 지문이 다르면 읽기 위치·북마크·저장 인코딩을 초기화한다.

수동 인코딩 선택은 디코딩 요청에만 전달하고 성공 전에는 저장소를 변경하지 않는다. 저장 실패 시 뒤로가기와 명시적 앱 종료는 실행하지 않아 사용자가 재시도할 수 있게 한다.

Android CP949는 전체 문자열 변환 경로이므로 32MiB로 제한한다. UTF-8과 스트리밍 UTF-16은 기존 64MiB 제한을 유지한다.

## 복구와 백업

`AppStore.load()`는 현재 실행에서 새로 생성한 백업뿐 아니라 가장 최근의 `state.json.broken.*`도 찾는다. 복구 배너는 재시작 후에도 표시한다.

배너에는 다음 동작을 제공한다.

- 손상 백업 내보내기: Android `ACTION_CREATE_DOCUMENT`를 통해 사용자가 고른 위치로 스트리밍 복사한다.
- 수정한 상태 가져오기: 기존 JSON 가져오기 검증을 사용한다.
- 성공한 복구 뒤 내부 손상 백업 삭제: 다음 실행에서 같은 배너가 반복되지 않게 한다.

개인정보 설명과 맞도록 Android Auto Backup을 비활성화한다. 재생성 가능한 페이지 인덱스는 Application Support가 아니라 임시 캐시 디렉터리에 저장한다.

## 페이지 처리

페이지 계산 호출을 하나의 오류 처리 함수로 감싼다. 실패하면 페이지 모드에는 오류와 재시도 버튼을 표시하고, 스크롤 모드는 본문을 유지한 채 메시지만 표시한다. 재시도 전 코디네이터의 실패한 계산 키를 초기화한다.

`PageTurnView`는 현재 페이지나 방향이 바뀔 때만 상호작용을 초기화한다. 진행 중 색인이 뒤에 페이지를 추가해 `itemCount`만 증가하는 경우 현재 드래그를 유지한다.

## 브랜딩과 호환성

다음을 `fantextviewer`/`판갤텍뷰`로 변경한다.

- Dart 패키지 이름과 `package:` import
- 앱 클래스와 사용자 표시 문자열
- Android namespace와 Kotlin 패키지
- 내부 MethodChannel 이름
- 신규 로컬 키 파일명·별칭과 Android smoke 테스트 명칭
- README와 현재 제품 설명

다음은 호환성을 위해 유지한다.

- `applicationId=com.songs.geulbom`
- 기존 `key.properties`가 참조하는 기존 키 파일과 별칭
- 저장소 디렉터리 자체와 과거 설계 문서의 역사적 명칭

새 키 생성 스크립트는 `fantextviewer-local.jks`를 만들지만 기존 사용자의 `key.properties`에는 변경을 강제하지 않는다.

README는 기존 문장을 부분 교정하지 않고 현재 동작을 기준으로 짧게 다시 정리한다. 다음을 정확히 명시한다.

- 판갤텍뷰의 현재 읽기·검색·북마크·페이지 기능
- 내부 공유 저장소 폴더 탐색에는 모든 파일 접근 권한이 필요함
- SD·USB·클라우드 파일은 단일 파일 가져오기로 앱 내부에 복사됨
- UTF-8·UTF-16은 64MiB, Android CP949는 32MiB까지 지원함
- 상태와 가져온 파일은 앱 내부에 저장되고 Android Auto Backup은 비활성화됨
- 기존 설치 업데이트에는 레거시 `applicationId`와 동일한 서명 키가 필요함
- 현재 Flutter/Dart/Android 요구 버전과 로컬·CI 검증 명령

## 검증

각 동작은 실패하는 회귀 테스트를 먼저 추가하고 최소 구현으로 통과시킨다.

- 시스템 선택 파일이 캐시 경로가 아닌 앱 소유 영구 경로가 되는 Android 계측 테스트
- 기존 캐시 파일 승격과 문서 상태 경로 이동 테스트
- 읽기 성공·저장 실패에서도 본문이 표시되는 위젯 테스트
- 실패한 수동 인코딩이 저장되지 않는 테스트
- 같은 크기·수정일이지만 내용 지문이 다른 파일의 상태 초기화 테스트
- 손상 백업 재발견·내보내기·복구 완료 정리 테스트
- 페이지 계산 실패 후 재시도 테스트
- 색인 중 `itemCount` 증가가 스와이프를 취소하지 않는 테스트
- CP949 32MiB 제한 테스트
- 브랜딩 문자열과 Android 설정 검사

완료 기준은 `dart format`, `flutter analyze`, 전체 `flutter test`, Android 릴리스 APK 빌드가 모두 성공하고 기존 `applicationId`가 유지되는 것이다.
