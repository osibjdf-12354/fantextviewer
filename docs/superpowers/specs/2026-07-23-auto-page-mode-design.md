# 오토 페이지 모드 설계

## 목표

독자가 지정한 간격마다 다음 페이지로 자동 이동하는 오토모드를 추가한다. 오토모드는 기본 OFF이며 햄버거 메뉴에서 켜고 끈다. 메뉴, 설정, 대화상자 또는 앱 비활성 상태에서는 자동 이동하지 않는다.

## 설정과 실행 상태

- `ReaderSettings`에 `autoPageIntervalSeconds`를 추가한다.
- 기본값은 5초, 허용 범위는 1~60초, 조절 단위는 1초다.
- 마지막으로 선택한 간격은 저장하고 앱을 다시 실행해도 유지한다.
- 범위 밖이거나 손상된 저장값은 5초로 복구한다.
- 오토모드 ON/OFF는 저장하지 않는 `ReaderView` 실행 상태다. 새 리더 화면과 앱 재실행은 항상 OFF로 시작한다.

## 읽기 방식 전환

- 오토모드를 켜면 실제 저장 설정을 변경하지 않고 유효 읽기 방식을 `스와이프`, 넘김 방향을 `상하`로 임시 재정의한다.
- 오토모드를 끄면 켜기 전 저장 설정으로 즉시 돌아간다.
- 오토모드 중 표시 설정에서 읽기 방식이나 방향을 변경할 수 있다. 오토모드가 켜져 있는 동안에는 상하 스와이프를 유지하고, 끄면 새로 저장된 설정을 사용한다.

## UI

- 햄버거 메뉴 상단에 `오토모드` 스위치를 추가한다.
- 표시 설정에 `오토 페이지 간격` 단계 조절 항목을 추가한다.
- 간격 조절 항목 아래에 다음 도움말을 표시한다.

  `세로 스크롤에서도 오토모드를 켜면 스와이프·상하 넘김으로 자동 전환됩니다.`

- 표시 설정의 기존 초안 적용 방식을 유지한다. 간격 변경은 설정 화면을 닫을 때 한 번 적용한다.
- 오토모드가 켜진 상태에서 간격이 바뀌면 설정 화면을 닫은 뒤 새 간격을 처음부터 계산한다.

## 타이머와 페이지 이동

- `ReaderView`가 1회성 타이머 하나를 소유한다.
- 타이머 만료 시 `PageTurnView`의 기존 180ms 애니메이션으로 다음 페이지를 상하 방향으로 넘긴다.
- 페이지 이동이 끝난 시점부터 새 간격을 계산한다.
- 사용자가 직접 탭 또는 스와이프로 페이지를 넘기면 오토모드를 유지하고 해당 페이지부터 간격을 다시 계산한다.
- 사용자가 페이지에 손을 대고 있는 동안에는 타이머가 만료되지 않도록 취소하고, 상호작용이 끝나면 다시 계산한다.
- 다음 페이지가 아직 계산되지 않았으면 기다린다. 페이지 계산 결과에 다음 페이지가 추가되면 그때 새 타이머를 시작한다.
- 현재 페이지의 원문 끝이 문서 끝과 같으면 실제 마지막 페이지다. 마지막 페이지에 도달하면 오토모드를 OFF로 바꾸고 종료 안내를 표시한다.

## 일시정지

다음 상태에서는 타이머를 취소하고 페이지를 넘기지 않는다.

- 햄버거 메뉴가 열려 있음
- 위치 이동, 본문 검색, 북마크, 표시 설정 또는 파일 정보 화면이 열려 있음
- 앱이 백그라운드 또는 비활성 상태
- 페이지 터치·스와이프·탭 상호작용 중

오토모드가 여전히 ON이고 모든 정지 조건이 해제되면 전체 간격을 처음부터 다시 계산한다. 햄버거 메뉴 안에서 ON으로 바꾼 경우에도 메뉴를 닫은 뒤 시작한다.

## 구현 범위

- `ReaderSettings`: 간격 저장, 복원, 복사, 유효성 처리
- `ReaderView`: 비저장 ON/OFF 상태, 단일 타이머, 일시정지 조건, 임시 읽기 방식, 마지막 페이지 처리
- `PageTurnView`: 기존 애니메이션을 재사용하는 자동 다음 페이지 신호와 사용자 상호작용 알림
- 표시 설정과 햄버거 메뉴 UI

별도의 전역 서비스, 남은 시간 표시, 반복 재생, 오토모드 상태 저장은 추가하지 않는다.

## 검증

- 간격 기본값, 저장 왕복, 범위 밖 값의 5초 복구
- 새 리더 화면에서 오토모드가 항상 OFF
- ON 시 스와이프·상하 넘김으로 임시 전환하고 OFF 시 저장 설정 복원
- 설정된 시간 뒤 기존 세로 애니메이션으로 한 페이지 이동
- 수동 페이지 이동 뒤 타이머 초기화
- 햄버거 메뉴, 각 모달 화면, 앱 비활성 상태에서 정지하고 닫힌 뒤 재시작
- 다음 페이지 계산 대기와 실제 마지막 페이지에서 OFF·안내
