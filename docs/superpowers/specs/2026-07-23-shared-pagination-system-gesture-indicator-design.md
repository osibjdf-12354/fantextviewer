# 통합 페이지 계산 및 시스템 제스처 영역 표시 설계

## 목표

- 세로 스크롤, 스와이프, 탭, 오토모드가 같은 페이지 경계와 전체 페이지 수를 사용한다.
- 오토모드를 켤 때 현재 읽던 본문과 페이지가 바뀌지 않는다.
- 페이지 숫자를 본문 위가 아닌 시스템 제스처 영역의 우하단에 표시한다.
- 시스템 안전영역이나 고정 높이로 본문을 불필요하게 줄이지 않는다.

## 현재 문제

긴 문서의 세로 스크롤 모드는 페이지 계산을 생략하고 글자 수 기반 추정값을 사용한다. 스와이프·탭·오토모드로 전환하면 실제 페이지 계산값으로 바뀌어 페이지 번호가 수백 장 이동한 것처럼 보인다.

본문 전체를 `SafeArea`로 감싸고 페이지 표시 공간을 고정값으로 빼고 있어 화면이 좁다. 페이지 표시는 본문 위에 겹치며, 전환 프레임에서 글씨가 시스템 영역에 잠깐 노출될 수 있다.

## 선택한 설계

### 페이지 계산

- 모든 읽기 모드에서 기존 `paginateText` 결과와 페이지 캐시를 공유한다.
- 긴 문서의 세로 스크롤 모드도 동일한 페이지 계산을 백그라운드에서 시작한다.
- 계산된 페이지가 현재 오프셋까지 도달하기 전에는 추정 페이지 숫자를 표시하지 않고 `계산 중`을 표시한다.
- 계산이 완료되면 세로 스크롤의 현재 오프셋을 같은 페이지 목록에 매핑한다.
- 오토모드 전환은 페이지를 다시 추정하지 않고 현재 오프셋을 공유 페이지 목록에 매핑한다.
- 비동기 위치 계산 중에는 이전 페이지 창을 숨겨 잘못된 본문이 한 프레임이라도 보이지 않게 한다.

### 화면 배치

- 리더 본문의 `SafeArea`와 고정 페이지 표시 여백을 제거한다.
- 시스템 제스처 영역 높이는 `MediaQuery.viewPadding.bottom`에서 가져온다.
- 본문 영역과 시스템 제스처 영역을 겹치지 않는 레이아웃으로 나눈다.
- 페이지 숫자는 시스템 제스처 영역의 우하단에 표시한다.
- 페이지 계산에는 실제 본문 영역의 크기를 그대로 사용한다. 별도의 고정 높이를 빼지 않는다.

## 상태 및 예외

- 시스템 제스처 영역이 없는 환경에서는 페이지 표시가 본문 하단 우측에 겹치지 않도록 표시 자체 높이만 사용한다.
- 빈 문서는 기존 빈 파일 화면을 유지한다.
- 페이지 캐시가 있으면 즉시 같은 페이지 계산 결과를 모든 모드에서 사용한다.
- 캐시가 없으면 계산 중 상태를 표시하고, 완료 뒤 숫자로 교체한다.

## 검증

- 긴 문서의 세로 스크롤과 스와이프·탭이 같은 현재/전체 페이지를 표시한다.
- 먼 위치에서 오토모드를 켜도 저장 오프셋과 표시 본문이 유지된다.
- 새 페이지 창 계산 중 이전 위치의 본문이 노출되지 않는다.
- 하단 시스템 인셋이 바뀌어도 고정값 없이 본문과 페이지 표시가 분리된다.
- 페이지 숫자가 시스템 제스처 영역 우하단에 배치된다.
- 전체 Flutter 분석과 테스트를 통과한다.

## 제외 범위

- 세로 스크롤 동작을 페이지 단위 스크롤로 변경하지 않는다.
- 페이지 계산 알고리즘이나 두 줄 겹침 규칙은 변경하지 않는다.
- 시스템 상단 상태바와 다른 화면의 안전영역은 변경하지 않는다.
