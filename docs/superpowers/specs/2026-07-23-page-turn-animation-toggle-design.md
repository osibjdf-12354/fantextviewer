# 페이지 넘김 애니메이션 설정 설계

## 목표

표시 설정에서 페이지 넘김 애니메이션을 켜거나 끌 수 있게 한다. 기본값은 켜짐이며 앱을 다시 실행해도 선택값을 유지한다.

## 동작

- 설정 이름은 `페이지 넘김 애니메이션`이고 스위치로 제공한다.
- 설정 변경은 기존 표시 설정과 동일하게 창을 닫을 때 적용한다.
- 켜짐에서는 현재 스와이프 추적과 전환 애니메이션을 그대로 유지한다.
- 꺼짐에서는 드래그 중 페이지가 움직이지 않는다.
- 꺼짐에서도 기존 거리·속도 기준을 만족한 스와이프를 끝내면 페이지를 즉시 교체한다.
- 탭, 오토모드, 접근성 동작도 애니메이션 없이 즉시 교체한다.
- 짧은 움직임, 범위를 벗어난 넘김, 긴 누르기와 다중 터치는 기존처럼 페이지를 바꾸지 않는다.

## 구조

`ReaderSettings`에 기본값이 `true`인 불리언 설정을 추가하고 기존 JSON 및 `copyWith` 흐름으로 저장한다. `ReaderView`는 이 값을 `PageTurnView`에 전달한다.

`PageTurnView`는 설정이 켜졌을 때 기존 경로를 그대로 사용한다. 꺼졌을 때는 드래그 진행률을 화면 변환에 반영하지 않고, 유효한 넘김이 확정되는 시점에 `onPageChanged`를 바로 호출한다. 별도의 페이지 뷰나 새 애니메이션 구현은 만들지 않는다.

## 검증

- 설정이 없던 기존 저장 데이터에서 기본값이 켜짐인지 확인한다.
- 설정값의 JSON 왕복과 `copyWith` 보존을 확인한다.
- 표시 설정 스위치가 현재값을 표시하고 닫을 때 저장하는지 확인한다.
- 꺼짐에서 드래그 중 본문 위치가 고정되고, 유효한 스와이프·탭·프로그램 전환이 즉시 페이지를 바꾸는지 확인한다.
- 켜짐의 기존 페이지 전환 테스트가 계속 통과하는지 확인한다.
