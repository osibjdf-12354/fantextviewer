# 글봄 Android 텍스트 뷰어 설계

## 목표

개인용 Android APK로 설치하여 기기의 `.txt` 파일을 찾아 읽고, 세로 스크롤과 화면 단위 좌우 넘김을 전환하며, 읽던 위치와 북마크를 자동 보존한다. 한국어 텍스트는 UTF-8, UTF-16, CP949/EUC-KR을 지원한다.

## 범위

- 앱 표시 이름은 `글봄`, Flutter 프로젝트와 Git 저장소 이름은 `geulbom`이다.
- Android만 지원한다.
- 기기 저장소의 폴더와 `.txt` 파일을 앱 안에서 탐색한다.
- 최근 파일, 파일명 검색과 정렬을 제공한다.
- 세로 스크롤과 화면 단위 좌우 넘김을 읽는 도중 전환할 수 있다.
- 파일별 읽던 위치, 북마크 여러 개, 선택한 인코딩을 자동 저장한다.
- 본문 검색, 위치 이동, 표시 설정과 화면 켜짐 유지를 제공한다.
- 파일 편집, 클라우드 동기화, PDF/EPUB 지원은 포함하지 않는다.

## 저장소 접근

개인용 APK이므로 Android의 전체 파일 접근 권한으로 공유 저장소를 직접 탐색한다. 최초 실행 시 권한 필요성과 설정 이동을 한국어로 설명한다. 사용자가 권한을 허용하지 않으면 시스템 파일 선택기로 `.txt` 파일 하나를 열 수 있게 한다. 권한 실패가 앱 전체의 실행 실패로 이어지지 않게 한다.

탐색기는 폴더를 먼저, 파일을 나중에 표시하며 이름순과 수정일순을 지원한다. 기본 목록에는 `.txt`만 보이고 숨김 파일은 제외한다. 최근 파일에서 삭제되거나 이동된 경로를 열면 목록에서 제거할지 안내한다.

## 화면 구조

### 홈과 파일 탐색기

- 상단에는 현재 경로, 뒤로 가기, 검색, 정렬 메뉴가 있다.
- 첫 화면에는 최근 읽은 파일과 저장소 진입점을 표시한다.
- 폴더 행을 누르면 이동하고 `.txt` 행을 누르면 읽기 화면을 연다.
- 빈 폴더, 권한 없음, 읽기 실패를 각각 구분해 한국어로 표시한다.

### 읽기 화면

- 상단바의 햄버거 버튼은 파일 열기, 위치 이동, 본문 검색, 북마크, 표시 설정, 파일 정보로 연결된다.
- 상단바에는 현재 파일명과 북마크 추가 버튼도 둔다.
- 세로 스크롤 모드에서는 아래로 이어 읽는다.
- 페이지 모드에서는 좌우 스와이프로 화면 단위 페이지를 넘긴다.
- 페이지 모드 하단에는 현재/전체 페이지와 이동 슬라이더를 표시한다.
- 스크롤 모드 하단에는 현재 진행률을 작게 표시한다.

### 설정

설정값은 읽기 모드, 글자 크기, 줄 간격, 좌우 여백, 밝은색·어두운색·세피아 테마, 화면 켜짐 유지다. 한국어 글꼴은 Android 시스템 기본 글꼴을 사용하여 별도 폰트 파일과 APK 용량을 추가하지 않는다.

## 텍스트 처리와 한글 호환성

파일 바이트의 BOM을 먼저 확인하고 UTF-8을 엄격하게 시도한다. 실패하면 CP949/EUC-KR로 해석한다. UTF-16 LE/BE도 BOM으로 판별한다. 자동 판별 결과가 틀리면 파일 정보 메뉴에서 UTF-8, UTF-16 LE, UTF-16 BE, CP949/EUC-KR 중 하나를 직접 선택하여 다시 연다. 파일별 수동 선택값은 다음 실행에도 유지한다.

디코딩 후 줄바꿈은 `CRLF`, `LF`, `CR`을 모두 `LF`로 정규화한다. 읽기와 검색은 디코딩된 Dart 문자열에서 수행한다.

## 위치, 페이지와 북마크

정본 위치는 디코딩된 문자열의 문자 오프셋이다. 페이지 모드에서는 각 페이지의 시작 오프셋을 저장한다. 세로 스크롤 모드에서는 화면에 처음 보이는 텍스트 구간의 시작 오프셋과 구간 내 위치를 저장한다. 앱이 백그라운드로 가거나 파일을 닫거나 진행 위치가 바뀔 때 지연 저장한다.

표시 폭, 높이, 글자 크기, 줄 간격 또는 여백이 바뀌면 페이지 경계를 다시 계산한다. 모드 전환과 재계산 뒤에는 저장된 문자 오프셋을 포함하는 페이지나 스크롤 구간으로 이동한다. 따라서 화면 구성 변화가 페이지 번호를 바꾸더라도 읽던 내용은 유지된다.

북마크는 파일 식별자, 문자 오프셋, 생성 시각, 해당 위치 주변의 짧은 문장을 저장한다. 북마크를 누르면 현재 모드에서 해당 오프셋으로 이동한다. 같은 위치의 중복 북마크는 만들지 않는다.

`특정 페이지로 이동`은 페이지 모드의 실제 페이지 번호를 사용한다. 스크롤 모드에서는 페이지 번호 또는 0~100% 진행률을 입력할 수 있으며, 페이지 번호를 입력하면 현재 표시 설정으로 계산된 해당 페이지 위치로 이동한다.

## 대용량 파일 처리

파일은 한 번 디코딩하되 화면에는 전체 문자열을 한 위젯으로 만들지 않는다. 스크롤 모드는 줄바꿈을 우선한 작은 텍스트 구간 목록을 사용한다. 페이지 모드는 Flutter `TextPainter`로 현재 화면에 맞는 페이지 시작 오프셋을 계산하고 결과를 메모리에 캐시한다. 전체 페이지가 필요한 이동 화면에서는 계산 진행 상태를 표시한다.

첫 버전은 일반적인 소설 텍스트 파일을 대상으로 한다. 수십 MB를 크게 넘는 파일도 강제로 차단하지 않지만, 메모리 부족 위험이 있는 매우 큰 파일에는 경고를 표시한다. 스트리밍 디코더와 디스크 페이지 색인은 실제 사용에서 필요성이 확인될 때 추가한다.

## 상태와 영속 저장

별도 상태 관리 프레임워크 없이 Flutter의 `ChangeNotifier`와 `ListenableBuilder`를 사용한다. 앱 상태는 파일 탐색 상태와 현재 문서 상태만 유지한다.

로컬 설정 저장소에는 다음을 JSON 호환 값으로 저장한다.

- 전역 표시 설정과 마지막 읽기 모드
- 최근 파일 경로와 마지막 연 시각
- 파일별 문자 오프셋, 스크롤 세부 위치, 수동 인코딩
- 파일별 북마크 목록

파일 식별자는 정규화된 절대 경로를 기본으로 하고 파일 크기와 수정 시각을 함께 확인한다. 같은 경로의 파일 내용이 교체되면 이전 위치가 파일 길이를 넘지 않도록 보정한다.

## 오류 처리

- 권한 거부: 권한 설정으로 이동하는 버튼과 시스템 파일 선택기 버튼을 제공한다.
- 파일 삭제·이동: 최근 목록을 정리할 수 있게 하고 탐색기로 돌아간다.
- 디코딩 실패: 인코딩 수동 선택 화면을 연다.
- 읽기 중 I/O 실패: 읽던 위치를 보존하고 재시도 또는 탐색기로 돌아가기를 제공한다.
- 빈 파일: 빈 파일임을 표시하되 정상적으로 열고 북마크와 검색은 비활성화한다.
- 페이지 계산 실패: 읽기 위치를 보존하고 세로 스크롤 모드로 전환할 수 있게 한다.

## 검증

- 단위 테스트: BOM/UTF-8 판별, 줄바꿈 정규화, 텍스트 구간 분할, 위치 보정, 북마크 중복 방지, 저장 모델 직렬화
- 위젯 테스트: 파일 목록 필터·정렬, 읽기 모드 전환, 햄버거 메뉴, 위치 이동 입력 검증, 빈 파일과 오류 화면
- 수동 확인용 샘플: UTF-8, UTF-16 LE/BE, CP949로 인코딩된 한국어 `.txt`
- 완료 조건: `flutter test`, `flutter analyze`, `flutter build apk --debug`, `flutter build apk --release`가 모두 성공하고 실제 Android에서 권한 승인, 파일 열기, 재실행 후 위치 복원, 북마크 이동을 확인할 수 있다.

## 구현 원칙

한 기능만 위한 추상화나 별도 데이터베이스는 만들지 않는다. Flutter와 Dart 표준 기능을 우선하며 Android 권한, 시스템 파일 선택, CP949 변환, 간단한 설정 저장, 화면 켜짐 유지에 필요한 검증된 패키지만 사용한다. 모든 저장은 기기 안에서만 수행하고 네트워크 권한은 요청하지 않는다.
