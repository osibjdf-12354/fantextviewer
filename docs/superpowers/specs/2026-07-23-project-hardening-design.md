# 글봄 프로젝트 안정화 설계

## 목표

현재 동작과 사용자 설정을 유지하면서 저장 안전성, Android 파일 접근, 대용량 문서 성능, 읽기 위치 정확도, 접근성, 배포 신뢰성을 개선한다. `ReaderView`에 집중된 상태와 부수 효과는 Flutter 기본 `ChangeNotifier` 기반 컨트롤러로 옮긴다. 이 앱 규모에서 별도 상태관리 패키지는 이점보다 마이그레이션 비용과 결합도 증가가 크므로 추가하지 않는다.

## 구조

- `ReaderScreen`은 파일 로딩과 오류 화면 전환만 담당한다.
- `ReaderController`는 읽기 설정, 위치, 검색, 자동 넘김과 저장 예약을 소유한다.
- `ReaderPaginationCoordinator`는 화면 크기에 종속된 페이지 계산, 캐시, 먼 페이지 이동과 계산 세대를 소유한다.
- `ReaderView`는 컨트롤러를 구독해 화면을 그리며 입력을 명령으로 전달한다.
- 설정 패널은 별도 `ReaderSettingsSheet` 위젯으로 분리한다.
- 기존 `AppStore`, 텍스트 디코더, 페이지 캐시는 작은 서비스로 유지하고 불필요한 저장소 추상화는 만들지 않는다.

## 데이터 안전성

- 앱 데이터에 명시적 스키마 버전을 추가하고 구버전 JSON을 단계적으로 마이그레이션한다.
- 숫자와 열거형 입력을 검증하고 안전한 범위로 보정한다.
- 저장 요청은 직렬화하고, 요청 시점의 불변 스냅샷을 임시 파일에 쓴 뒤 원본과 원자적으로 교체한다.
- 손상 파일 백업은 덮어쓰지 않도록 시각이 포함된 이름을 사용하며 복구 실패를 진단 가능한 오류로 남긴다.
- 문서 상태에는 파일 크기와 수정 시각으로 만든 지문을 저장한다. 파일이 바뀌면 인코딩, 읽기 위치, 북마크와 페이지 캐시를 안전하게 초기화한다.
- 앱 종료와 수명주기 변경 때 대기 중인 저장을 명시적으로 완료한다.

## Android 파일 및 서명

- `MANAGE_EXTERNAL_STORAGE`, 레거시 저장소 권한, `permission_handler`를 제거한다.
- 기존 `file_selector`의 시스템 폴더 선택기를 사용해 사용자가 허용한 위치만 탐색한다.
- 개인 사이드로드 릴리스는 로컬 `key.properties`와 비추적 키스토어를 사용한다. 키가 없으면 릴리스 빌드는 설명 가능한 오류로 중단하고, 로컬 키를 만드는 PowerShell 스크립트를 제공한다.
- CI는 저장소에 키를 넣지 않고 실행 중 임시 키를 생성해 릴리스 빌드까지 검증한다.

## 문서 처리와 읽기 위치

- 파일 크기는 읽기 전에 검사한다. 지원 한도를 넘는 파일은 메모리에 올리지 않고 안내한다.
- UTF-8은 스트림 디코딩을 사용하고, UTF-16/CP949는 명확한 상한 안에서만 전체 디코딩한다.
- 텍스트 조각은 줄 경계와 유니코드 문자 경계를 보존한다. 선택 영역을 조각 전체에 공유해 경계 선택을 가능하게 한다.
- 스크롤 위치는 조각 시작점이 아니라 문자 오프셋과 조각 내 정렬값을 함께 저장하고 복원한다.
- 스크롤 모드에서는 현재 위치 주변 페이지만 우선 계산한다. 전체 페이지 수 표시나 먼 위치 이동이 필요할 때 나머지를 낮은 우선순위로 계산한다.
- 페이지 계산 진행률은 매 배치마다 전체 화면을 다시 만들지 않고 필요한 UI만 갱신한다.

## 보조 기능

- 검색은 컨트롤러에서 실행하고 다음/이전 결과 이동과 현재 결과 강조를 제공한다.
- 폴더 목록 비동기 결과에는 세대 번호를 적용해 늦게 끝난 이전 요청이 최신 화면을 덮지 못하게 한다. 개별 파일 메타데이터 오류는 해당 항목만 제외한다.
- 글꼴은 확장자뿐 아니라 크기와 SFNT 헤더를 검사한다. 런타임에 이미 로드된 글꼴 삭제의 의미를 UI에 명확히 하고 재시작 전 상태가 꼬이지 않게 한다.
- 시스템의 애니메이션 줄이기 설정을 페이지 전환과 자동 넘김에 반영한다.
- 낮은 대비 경고에는 즉시 기본 색으로 복구하는 동작을 제공한다.
- 한국어 전용 제품이라는 범위를 README에 명시하고 사용자 문자열은 한 파일로 모아 향후 지역화 경계를 만든다.

## 정리와 검증

- 사용하지 않는 `cupertino_icons`, 중복 인코딩 판별 API, 완료된 과거 구현 계획을 제거한다.
- Kotlin Gradle 플러그인 적용 경고가 사라지도록 최신 직접 의존성과 Flutter의 내장 Kotlin 구성으로 이동한다.
- 저장 경쟁, 손상 복구, 마이그레이션, 잘못된 JSON, 파일 교체, 폴더 탐색 경쟁, 실제 64KiB 무개행 텍스트, 정확한 위치 복원, 큰 파일 제한, 글꼴 검증, reduced motion을 회귀 테스트한다.
- GitHub Actions에서 포맷 검사, 정적 분석, 전체 테스트, Android 릴리스 빌드를 실행한다.
- 최종 검증은 `dart format`, `flutter analyze`, `flutter test`, `flutter build apk --release`와 실제 APK 산출물 확인으로 구성한다.
