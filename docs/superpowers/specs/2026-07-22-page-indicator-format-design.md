# 페이지 표시 형식 설정 설계

## 목표

읽기 화면 하단의 `페이지` 문구를 제거하고, 사용자가 `현재 페이지` 또는 `현재/전체 페이지` 형식을 선택할 수 있게 한다. 햄버거 메뉴의 현재 페이지 문구는 유지하고 북마크 관련 페이지 표시는 숫자만 사용한다.

## 설정과 호환성

`ReaderSettings`에 `showTotalPages` 불리언을 추가한다. 기본값은 `false`이며 JSON에 저장한다. 기존 설정 파일에 필드가 없으면 `false`로 복원하므로 기존 사용자에게는 현재 페이지만 숫자로 표시된다.

표시 설정 바텀시트에 `페이지 표시` 영역을 추가하고 다음 두 `ChoiceChip`을 제공한다.

- `현재 페이지만`: 하단에 `3`처럼 표시
- `현재/전체 페이지`: 하단에 `3/120`처럼 표시

설정은 기존 `적용` 버튼과 `AppStore.updateSettings` 저장 흐름을 그대로 사용한다.

## 표시 범위

- 읽기 화면 하단: 선택한 형식에 따라 `현재` 또는 `현재/전체` 숫자만 표시한다.
- 전체 페이지 계산 중: 기존 추정 전체 페이지 수를 사용하며, 정확한 계산이 끝나면 자동으로 갱신한다.
- 햄버거 메뉴: 기존 `현재 3페이지` 문구를 유지한다.
- 북마크 저장 안내: `3에 북마크를 저장했습니다.`처럼 표시한다.
- 북마크 목록: 페이지 번호를 `3`처럼 표시한다.
- 위치 이동, 계산 안내, 오류 메시지 등 페이지 의미를 설명해야 하는 다른 문구는 변경하지 않는다.

## 오류 처리

새 설정은 불리언이며 별도 입력 검증이 필요 없다. 손상되거나 누락된 JSON 값은 기존 역직렬화 방식대로 기본값 `false`를 사용한다.

## 테스트

- `showTotalPages` JSON 저장·복원과 누락 필드 기본값을 검증한다.
- 기본 하단 표시가 숫자만 포함하는지 검증한다.
- 표시 설정에서 `현재/전체 페이지`를 선택하고 적용하면 `현재/전체` 형식이 표시되고 저장되는지 검증한다.
- 햄버거 메뉴의 `현재 N페이지`가 유지되는지 검증한다.
- 북마크 저장 안내와 목록에서 `페이지` 문구가 제거되는지 검증한다.

## 제외 범위

새 페이지 계산 방식, 별도 포맷터 추상화, 추가 의존성은 도입하지 않는다.
